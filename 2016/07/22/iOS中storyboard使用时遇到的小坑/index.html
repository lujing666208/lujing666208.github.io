iOS中storyboard使用时遇到的小坑

今天在写demo时，遇到两个小坑，mark一下，以防下次再掉进去。

1.iOS7以后使用tableView时，上面会有空白

在有导航栏存在时，tableView上方会有空白，这是因为iOS7在Controller中新增了这个属性：
automaticallyAdjustsScrollViewInsets，当设置为true时（默认true），如果视图里面存在唯一一个UIScrollView或其子类View，那么它会自动设置相应的内边距，这样可以让scroll占据整个视图，又不会让导航栏遮盖。

2.storyboard设置错误造成的崩溃：”this class is not key value coding-compliant for the key …”问题

在storyboard中对控件进行代码关联时，第一次属性名写的不合适，在代码中改了名字，然后又拖了一次线到新属性上，运行程序，跳转到此控制器时，程序崩溃，报错为"this class is not key value coding-compliant for the key ..."。查了半天，把该屏蔽的代码都屏蔽了，还是崩溃，感觉整个人都不好了。。。最后重整心情，从根源查找，既然说某个key不能被设置，那就看看哪里有用到这个key,代码中没有，就到storyboard中查找，突然发现，这个控制器上一个控件关联了代码中的两个属性，分别是之前设置的错误名字和新名字，其中错误名字正是报错中所说的key,而它在代码中已经不存在了，所以运行时，没法通过KVC设置这个key的值。

解决方法

删除控件和不存在的属性之间的关联

今天被这两个坑耽误了好长时间，感觉还是对storyboard使用不熟，一旦出现细节上的错误，查找就很耗费时间。所以记录下来以备用。
